## Market Regime · A股/港股市场状态识别模块 — 需求文档

> [!info] 这份文档解决什么
> 把原文《市场状态识别的混合机器学习实战》的混合 ML 框架（**无监督发现 + 监督预测**）从美股单市场迁移到 **A 股 + 港股**，作为 **binwh.QuantPilot 的一个模块 `market_regime`** 实现。本文是 brainstorm 产出，回答 **做什么**；具体怎么做（技术选型、文件结构、API 设计）留给后续 plan。

---

## 一、背景与问题

原文章用一篮子美股宏观 ETF（股票/高收益债/投资级债/VIX）演示了「无监督聚类发现状态 + 监督分类预测状态」的两阶段范式，输出 ROC-AUC≈1.0。

迁移到 A 股/港股时，存在三个本质差异需要回应：

1. **市场结构不同**：A 股受国内信用/政策周期主导，港股受美元流动性/南向资金影响更大，两市场状态系统**不同步**。
2. **可用代理变量不同**：A 股/港股没有现成的 VIX，高收益信用利差数据也较难获取，需用替代代理。
3. **目标不同**：原文是研究 demo；本模块要产出**每周可信赖的状态视图 + 配置参考**，用于指导真实持仓，并接入 QuantPilot 现有架构。

### 与 QuantPilot 的关系

本模块**并入 `binwh.QuantPilot` 作为功能模块**，复用其 FastAPI 后端 + React 前端 + 数据层。QuantPilot 已有 `kline`/`stock` 行情数据基础，本模块在其上构建状态识别能力。

---

## 二、模块目标（第一版）

### 核心成果

**QuantPilot 内的一个「市场状态」功能模块**，每周（或按需）输出：

1. **A 股**当前所处状态（平静期 / 动荡期，可选扩展更多状态）+ 未来状态预测概率
2. **港股**当前所处状态 + 未来状态预测概率
3. **跨市场对比视图**（如「A 股平静 + 港股动荡」这种组合信号）
4. **历史状态回溯**（状态序列叠加在价格图上）
5. **定性配置方向**（攻 / 守 / 中性），不是精确权重

### 验收标准（第一版门槛）

> [!important] 核心门槛：状态切得对，而非指标漂亮
> **第一版的状态识别必须通过「人工对照历史重大阶段」校验**——把状态序列叠加在价格图上，人眼判断 2018 贸易战、2022、2024 等重大阶段的状态切换是否对得上宏观直觉。对不上就调，直到对得上为止。

辅助参考（不作为唯一门槛）：
- 轮廓系数 / KS 统计量 / 校准曲线等指标合理
- 状态分组的经济含义清晰（如动荡期日均收益为负、波动率显著高于平静期）

### 明确不在第一版目标内

见下方「四、范围边界」。

## 三、方案选择：方案 B（A股 + 港股并行 + 统一看板）

> [!note] 为什么不是 A 或 C
> - 方案 A（只做 A 股）：聚焦但缺「港股」这块核心诉求。
> - 方案 C（统一状态空间）：理论优雅但风险最高——两市场对共同驱动响应不同步，强行揉进一个状态空间易失真。
> - 方案 B：两市场独立建模更诚实，看板并排对比是原文没有的新价值，且易扩展。

**方案 B 要点：**

| 维度 | 决策 |
|---|---|
| 建模方式 | A 股、港股**各建一套独立状态模型**，不强行假设同步 |
| 呈现方式 | 两套模型输出在**同一个可视化页面**并排展示 |
| 扩展性 | 第三个市场（美股/黄金）可作为后续低成本扩展 |

## 四、范围边界

### 第一版 MVP 包含

- **数据层**：A 股（沪深300/中证500 + 国债 ETF/信用债代理 + 实现波动率）、港股（恒生指数/恒生科技 + 美债 10Y 代理 + 港股实现波动率）
- **特征工程**：多周期收益率、实现波动率、利差/代理变量，**严格用 expanding 窗口防数据泄漏**
- **状态识别**：PCA 降维 + 聚类（K-Means/GMM/HMM 由 plan 选型）发现状态 + 逻辑回归（或其他分类器）预测未来状态
- **可解释性验证**：人工对照历史阶段 + 混淆矩阵/KS/校准曲线
- **后端**：在 QuantPilot `backend/app/services/market_regime/` 实现核心逻辑，通过 `routers/market_regime.py` 暴露 API
- **前端**：React 页面展示两市场状态并排 + 跨市场对比 + 历史回溯
- **配置产出**：每个市场的攻/守/中性定性方向 + 跨市场组合的定性配置思路

### 明确不做（二期 / 未来再说）

- ❌ **状态→精确目标权重的自动配置**：需要状态条件下收益预测模型，是另一个建模问题，二期再做
- ❌ **实盘交易接入**：数据可来自 miniQMT，但第一版只读不写、不下单（与 QuantPilot 当前无交易模块一致）
- ❌ **「跑赢基准」作为验收门槛**：作为可选验证，不作为第一版门槛（避免过拟合回测陷阱）
- ❌ **统一状态空间建模**（方案 C 内容）
- ❌ **第三个市场**（美股/黄金）
- ❌ **公募基金参与状态判断**：基金只落配置端，不落状态特征端（净值是组合，信号被稀释）

## 五、关键决策与约束

| 决策点 | 结论 |
|---|---|
| 项目归属 | **并入 `binwh.QuantPilot` 作为 `market_regime` 模块** |
| 模块名 | `market_regime`（量化标准术语，与现有英文命名一致） |
| 第一版目标 | 状态识别准确可解释（不卡「跑赢基准」） |
| 方案 | B：A 股 + 港股各一套独立模型，统一看板 |
| 决策频率 | 周度中低频（偶尔周度操作） |
| 数据源 | 免费接口（akshare/Tushare）+ 券商 MCP + miniQMT（只读）+ 必要时抓取；尽量复用 QuantPilot 已有数据层 |
| 配置产出形态 | 定性方向（攻/守/中性），**非精确权重** |
| 基金/ETF 定位 | 落配置端，不落状态特征端 |
| 验收标准 | **人工对照历史阶段为核心门槛** + 指标辅助 |
| 防数据泄漏 | expanding 窗口标准化，不使用全样本统计量 |

## 六、已知假设与风险

> [!warning] 必须正视的风险
> 1. **标签泄漏陷阱**：原文 ROC-AUC≈1.0 几乎肯定有标签泄漏（用同期波动率聚类，再用同样特征预测同一标签）。本模块**第一版必须靠人工对照历史把关**，不能只看指标。
> 2. **A 股/港股代理变量可得性**：高收益信用利差、VIX 类波动率代理在 A 股/港股可能缺失，需用替代（如中证 500 实现波动率、美债 10Y 收益率代理离岸流动性）。具体代理选择由 plan 在实现时定，并记录其局限。
> 3. **港股状态特征源**：港股受美元流动性驱动显著，其特征空间可能与 A 股差异较大，两套模型的特征设计不一定对称。
> 4. **与 QuantPilot 数据层对接**：需确认 QuantPilot 现有 `kline`/`stock` 数据是否覆盖本模块所需标的（沪深300、中证500、恒生指数、国债 ETF 等），缺口部分由本模块自取或扩展数据层。
